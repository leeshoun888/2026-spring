# TU02 저화질 이미지 생성 (에이전트 2)

연결 문서: [[00_인덱스]] | [[TU01_이미지_프롬프트_작성]] | [[TU03_고화질_이미지_생성]]

## 중앙 프로세스

- 받는 것: 이미지 프롬프트 3개
- 처리: 프롬프트 3개를 병렬로 저화질 이미지 생성
- 내보내는 것: 저화질 프리뷰 이미지 3개
- 상태: UI 표시 후 `유저 응답 대기 중`

## 왼쪽 API 데이터 흐름

- 사용 API: 저화질 이미지 API
- 입력 데이터: 프롬프트 3개
- 출력 데이터: 프리뷰 이미지 3개
- 다음 전달: 선택된 이미지의 프롬프트를 TU03으로 익스포트

## 오른쪽 인터페이스 사용자 입력

- 화면 출력: 최신 배치 프리뷰 이미지 3개
- 입력 1(선택): 3개 중 1개 선택
- 입력 2(재생성): 새 프리뷰 배치 생성
- 입력 처리: 이전 배치는 숨기고 `inactive/archived`로 보관
