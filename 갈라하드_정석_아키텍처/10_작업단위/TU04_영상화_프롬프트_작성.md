# TU04 영상화 프롬프트 작성 (에이전트 4)

연결 문서: [[00_인덱스]] | [[TU03_고화질_이미지_생성]] | [[TU05_저화질_영상_생성]]

## 중앙 프로세스

- 받는 것: 확정된 고화질 이미지 1개 + 컷 정보
- 처리: 영상화 텍스트 프롬프트 3개 생성
- 내보내는 것: 영상화 프롬프트 3개
- 상태: 생성 완료 후 `유저 응답 대기 중`

## 왼쪽 API 데이터 흐름

- 사용 API: LLM 텍스트 API
- 입력 데이터: 첫 프레임 이미지 + 컷 목적
- 출력 데이터: 영상화 프롬프트 3개
- 다음 전달: 프롬프트 3개 + 기준 이미지를 TU05로 익스포트

## 오른쪽 인터페이스 사용자 입력

- 화면 출력: TU05의 저화질 영상 프리뷰와 연결된 프롬프트 후보
- 입력 1(선택): 선택 영상과 매칭된 프롬프트 1개 확정
- 입력 2(재생성): 새 프롬프트 3개 배치 생성
- 입력 처리: 이전 프롬프트 배치는 `inactive/archived` 보관
