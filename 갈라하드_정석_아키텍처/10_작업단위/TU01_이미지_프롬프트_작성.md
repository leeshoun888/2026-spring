# TU01 이미지 프롬프트 작성 (에이전트 1)

연결 문서: [[00_인덱스]] | [[TU02_저화질_이미지_생성]]

## 중앙 프로세스

- 받는 것: 콘티(컷 설명)
- 처리: 컷 1개당 이미지 프롬프트 3개 생성
- 내보내는 것: 이미지 프롬프트 3개
- 상태: 생성 완료 후 `유저 응답 대기 중`

## 왼쪽 API 데이터 흐름

- 사용 API: LLM 텍스트 API
- 입력 데이터: 컷 설명 텍스트
- 출력 데이터: 프롬프트 3개(배치)
- 다음 전달: TU02로 프롬프트 3개 익스포트

## 오른쪽 인터페이스 사용자 입력

- 화면 출력: 프롬프트 자체는 직접 노출하지 않고, TU02 미리보기 결과로 이어짐
- 입력 1(선택): 선택된 결과와 매칭된 프롬프트 1개만 active
- 입력 2(재생성): 새 프롬프트 3개 배치 생성
- 입력 처리: 미선택 프롬프트는 `inactive/archived` 보관
